String Buffer vs String Builder

공통점 : 둘다 문자열을 다루는 클래스. 문자열을 (유)동적으로 변경하는데 사용.

차이점 :
	  StringBuffer : 가변클래스. 스레드안전(thread-safe). 멀티스레드 환경에서 여러 스레드가 동시에 접근해도 데이터 무결성 유지.
	  				 동기화가 적용되어 성능면에서 Builder보다 느릴 수 있음.
	  				
	  StringBuilder : 가변클래스. 단일스레드(하나의 스레드)에서 사용하기에 적합. 문자열을 더하거나 수정할 때 메모리를 절약.
	  				  성능이 빠름. 동기화를 고려하지 않음. 스레드간에 동기화 관련 문제가 없음. 스레드간에 오버헤드가 없음.
	  				  *오버헤드(Overhead) : 어떤 작업을 수행할 때 추가적으로 필요한 자원, 시간, 비용 등.
